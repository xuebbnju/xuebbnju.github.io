Bootstrap知识复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，致敬页面知识点

1. 基础知识

  • Bootstrap 是移动设备优先的。
  • 栅格运用：最外层容器添加row类，每一列的div添加col-xs-*类，一共12列，*代表所占列数。
  • col-xs-*类下可以继续套col-xs-*类。
  • 自适应大小选择col-xs-*、col-sm-*、col-md-*。
  • col-sm-offset-*设置偏移的列数，可以用来实现居中。
  • container类。。。。。。

2. 用到的知识

  • Jumbotron类：顾名思义该组件可以增加标题的大小，并为登陆页面内容添加更多的外边距（margin）。使用超大屏幕（Jumbotron）的步骤如下：
    1. 创建一个带有 class .jumbotron. 的容器 <div>。
    2. 除了更大的<h1>，字体粗细 font-weight 被减为 200。
    3. 为了获得占用全部宽度且不带圆角的超大屏幕，请在所有的 .container class 外使用 .jumbotron class。
    4. 总而言之：增加标题字号，🏠外边距，背景颜色，以及圆角，不想要圆角就放在container类的外面。
  • thumbnail类：缩略图类，放在img标签外的a或者div标签中,会添加四个像素的内边距和一个灰色的边框，边框是圆角的（也可以放文字）。
  • 通过为 <img>元素添加以下相应的类，可以让图片呈现不同的形状。

二，个人主页知识点

1. 导航

  • 导航条是一个包含导航栏的容器，添加navbar类。类navbar-default必须有，用来添加默认导航条样式。类navbar-fixed-top/bottom设置导航条固定在顶部或底部。
  • 导航栏需要添加nav类，nav-tabs/nav-pills等用来设置不同的样式。如果放在了导航条中还必须添加navbar-nav类。

  • 当屏幕太小需要把导航栏折叠式，需要一个替代按钮，添加类navbar-toggle。并且为导航栏添加<nav></nav>容器,添加collapse和navbar-collapse。
  • 按钮属性设置data-toggle="collapse" data-target="#nav"，#nav为<nav></nav>容器id。

3. 几个高宽值

  • height：指元素内容的高度 ，jQuery中的height()方法返回的就是这个高度。
  • clientHeight：内容高度+padding高度 ，jQuery中的innerHeight()方法返回的就是这个高度。
  • offsetHeight：内容高度+padding高度+边框宽度 ，jQuery中的outerHeight()方法返回的就是这个高度。
  • width：指元素内容的宽度 ，jQuery中的width()方法返回的就是这个宽度。
  • clientWidth：内容高度+padding宽度 ，jQuery中的innerWidtht()方法返回的就是这个宽度。
  • offsetWidth：内容高度+padding高度+边框宽度 ，jQuery中的outerWidth()方法返回的就是这个宽度。

4. js和jQuery知识点

  • on()方法在被选元素及子元素上添加一个或多个事件处理程序。
  • 使用 on()方法添加的事件处理程序适用于当前及未来的元素（比如由脚本创建的新元素）。（动态的，简单的onclick都是静态的，由脚本创建的新元素不适用）。
  • 如需移除事件处理程序，请使用 off()方法。
  • 如需添加只运行一次的事件然后移除，请使用one() 方法:
    $(selector).on(event,childSelector,data,function,map)。
  • $(function(){}) 是 $(document).ready(function(){}) 的简写。

5. 即时搜索

概念： 用户输入一个字就去发请求匹配，相应的结果并显示出来。

  1. change事件，这个事件不太友好，触发必须满足两个条件：a）当前对象的值发生改变，并且是由
    键盘或鼠标事件激发的（脚本触发无效）；b）当前对象要失去焦点(onblur)

  2. keypress事件，这个一般，对比上面的要好，就是只能监听键盘事件，鼠标复制黏贴等操作就无能为力了

  3. propertychange和input事件，这两个事件其实是一回事，只是不同的标准罢了，可以同时绑定。

    • input是标准的浏览器（W3C）事件，一般应用于input元素，当input的value发生变化就会发生，无论是键盘输入还是鼠标,重点是黏贴的改变都能及时监听到变化；
    • propertychange是IE专属的事件，只要当前对象属性值发生改变就能触发。

6. event和this

  • event.targe是实际的触发事件的元素，而this是给哪个元素绑定了事件。
  • event.target代表当前触发事件的元素，可以通过当前元素对象的一系列属性来判断是不是我们想要的元素。
  • js中事件是会冒泡的，所以this是可以变化的，但event.target不会变化，它永远是直接接受事件的目标DOM元素；
  • this和event.target都是dom对象。
  • 如果要使用jquey中的方法可以将他们转换为jQuery对象。比如this和$(this)的使用、event.target和$(event.target)的使用；
