js之数组和字符串知识复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，数组

1. 基础概念

  • 数组就像是对象的一个子集
  • 有序，元素可以不同类型，是数组，也可以是对象 ,动态的无需指定大小。
  • 创建：字面量 不建议，长度0——2的23幂-1。new array构造器 new可以省略。
  • 与字符串的数据不可变不同，数组的数据是可变的，并且可以自由地改变。
  • 二维数组：数组的元素是数组。
  • 稀疏数组：并不含有0的连续索引，length比实际元素个数多 var arr=[,,](俩元素)，不常用，遍历用for in，for的话需要判断是否为undefined。

2. 常用操作

  • 元素删除：delete之后便undefined，长度不变 ,但依旧会占位置，但是用in这 个索引返回false，和arr[这个索引]=undefined是不一样的。
  • 尾部删除： 将arr.length减一，也会删除元素变成undefined ，arr.pop（）。
  • 头部删除：arr.shift()。
  • 尾部添加： arr[arr.Length]=4 等价于arr.push(4)。
  • 头部添加：arr.unshift()。
  • 遍历：for ，for in（受原型链影响，最好hasOwnpropotry,还有坑。）.
  • join方法：数组转为字符串，添加分隔符，实现重复元素的数组[a,a,a]。
  • reverse:倒序数组，会修改原数组
  • sort：排序，按字符串顺序排序，修改原数组，传一个函数当参数，实现数字排序
  • contact：数组合并，不修改原数组，传进去的参数是数组会被拉平，但只拉平第一次。
  • slice：返回片段，参数是首尾索引，左闭右开，负数表示倒数，只有一个的话就是到最后，不修改原数组
  • spice：数组拼接，修改原数组，保留剩下的，还可以添加，一个参数的话，删除从这个位置开始的片段，🏠一个参数是个数，后面的参数可以使删除位置补上的元素

3. IE9以上操作

  • for each：函数作为参数，值，索引。
  • map：映射，参数为函数，不修改原数组。
  • filter：函数作为参数，过滤删选，不修改原数组。
  • every：判断，是否每一个元素都符合这个条件，返回布尔值。
  • some：判断有一个就行，返回布尔值。
  • reduce：不修改原数组，返回计算结果。
  • reduceRight:从右到左的reduce。
  • indexof：返回子串位置，从左往右，第二个参数。
  • lastindexOf:右往左
  • Array.isArray([])，别的数组无法调用。

二，字符串

1. 基础概念

  • String 对象用于处理已有的字符块。
  • 在 JavaScript 中，字符串 的值是 不可变的，这意味着一旦字符串被创建就不能被改变。
  • 改变字符串的唯一方法是重新给它赋一个值。
  • 可以通过索引访问某个字符。
  • 如果把数字与字符串相加，结果将成为字符串。

2. 常用操作

  • charAt()：返回在指定位置的字符。
  • charCodeAt()：返回在指定的位置的字符的 Unicode 编码。
  • concat()：连接字符串，本身并没有被更改。
  • fromCharCode()：从字符编码创建一个字符串。String.fromCharCode(numX,numX,...,numX)。
  • indexOf():检索字符串位置。
  • lastIndexOf()：从后向前搜索字符串。
  • match()：可在字符串内检索指定的值，或找到一个或多个正则表达式的匹配，返回值。
  • replace()：用于在字符串中用一些字符替换另一些字符，或替换一个与正则表达式匹配的子串，返回一个新的字符串。
  • search() ：用于检索字符串中指定的子字符串，或检索与正则表达式相匹配的子字符串，返回索引。
  • slice() ：可提取字符串的某个部分，并以新的字符串返回被提取的部分，参数是起始索引。
  • sub()： 把字符串显示为下标。
  • substr()：从起始索引号提取字符串中指定数目的字符。
  • toLowerCase()： 把字符串转换为小写。
  • toUpperCase() ： 把字符串转换为大写。
  • toString() ： 返回原始字符串。
  • valueOf() ： 返回某个字符串对象的原始值。

js基础知识复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，数据类型

1. 分类

  • 原始类型：number,string , boollean 。
  • 合成类型：object (function,array,date，正则).
  • 特殊的值：null ,undefined.

2. 特性

  • JavaScript 拥有动态类型。这意味着相同的变量可储存不同类型的数据。
  • JavaScript 变量均为对象。当您声明一个变量时，就创建了一个新的对象。
  • 一切（引用类型）都是对象，对象是属性的集合。

3. undefined

  • 当 JavaScript中的变量被声明但没有初始化赋值的时候，程序内部会给它一个初始值 undefined。
  • 当你对一个值为undefined的变量进行运算操作的时候，算出来的结果将会是NaN，NaN 的意思是 “Not a Number”。
  • 当你用一个没有 定义 的变量来做字符串连接操作的时候，它会如实的输出undefined。

4. undefined和null

  • 最初设计:null是一个表示”无”的对象，转为数值时为0；undefined是一个表示”无”的原始值，转为数值时为NaN。
  • 目前用法：null表示”没有对象”，即该处不应该有值。undefined表示”缺少值”，就是此处应该有一个值，但是还没有定义。
  • undefined:
    1. 变量被声明了，但没有赋值时，就等于undefined。
    2. 调用函数时，应该提供的参数没有提供，该参数等于undefined。
    3. 对象没有赋值的属性，该属性的值为undefined。
    4. 函数没有返回值时，默认返回undefined。
  • null:
    1. 作为函数的参数，表示该函数的参数不是对象。
    2. 作为对象原型链的终点。

5. 两个“不变”的特性

  • 如果重新声明 JavaScript 变量，该变量的值不会丢失，在以下两条语句执行后，变量 carname 的值依然是 “Volvo”：var carname="Volvo";var carname;。
  • 在 JavaScript 中，字符串的值是不可变的，这意味着一旦字符串被创建就不能被改变。改变字符串 的唯一方法是重新给它赋一个值。

二，类型判断与检测

1. 判断相等

  1. 严格等于===:先判断类型，不同返回false，在判断值，null===null，undefind===undefind， 特殊：NaN与谁比较都不相等，即便是自己，对象比较其实比较的是引用，也不相等。
  2. 等于==:类型相同时和===相同。类型不同时会进行类型转换。
    • null==undefined；
    • string==num，string转换成数字；
    • Boolean==？，先转换成number，false：0，true：1；
    • 对象==number||string,object转换为基本类型；

2. 类型检测

  1. typeof:返回字符串，适合基本类型的检测和函数,对象。不能判断出数组和null，array和null返回object。

  2. instance of: 用于判断一个变量是否某个对象的实例。左操作数是对象，否则是false，右操作数是判断的类型，原型链，不同window下是不一样的。沿着A的proto这条线来找，同时沿着B的prototype这条线来找，如果两条线能找到同一个引用，即同一个对象，那么就返回true。如果找到终点还未重合，则返回false。

  3. object.prototype.toString.apply();

三，除类型外的一些概念

1. 表达式

  • 一些短语，会产生值。

2. 运算符

  • 普通运算符：一元，二元，三元；赋值运算符，比较运算符，算数运算符，位运算符，逻辑运算符，字符串运算符（字符串+）。
  • 特殊运算符：
    1. 条件运算符c?a:b;
    2. val=(1,2,3);val=3；从左到右依次计算，取最后的值。
    3. delete运算符，删除对象上的属性，变成undefined。
    4. in运算符，是否含有某一个key值。
    5. instanceof和typeof。
    6. new创建实例。
    7. this全局情况下指向window。
    8. void 不管后面的操作数是多少，都返回undefined 。void 0 //undefined；void (0) //undefined。

3. 语句

  • 块语句：没有块级作用域。比如for循环块语句。
  • 声明语句 ：var。
  • try catch语句：try后面必须接catch或finally.

  • with语句：修改作用域，不建议使用.
  • function声明语句：用来创建对象 ，函数声明前面可以调用语句，但是函数表达式前面不可以调用。

4. 一些区别

  1. 语句和表达式：

    • JavaScript中的表达式和语句是有区别的.一个表达式会产生一个值,它可以放在任何需要一个值的地方,比如,作为一个函数调用的参数。
    • 语句可以理解成一个行为.循环语句和if语句就是典型的语句。
    • 一个程序是由一系列语句组成的.JavaScript中某些需要语句的地方,你可以使用一个表达式来代替.这样的语句称之为表达式语句。
    • 但反过来不可以:你不能在一个需要表达式的地方放一个语句.比如,一个if语句不能作为一个函数的参数。

position复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，position：relative

  1. 没有完全脱离文档流，relative元素原本所占的空间仍保留。
  2. 元素框相对原来位置偏移某个距离，而不会影响其他元素的位置、大小的变化。
  3. 元素仍保持其未定位前的形状。
  4. z-index高于普通元素，会覆盖普通元素。
  5. position:relative天生可以限制position:absolute,作为position:absolute偏移的相对对象存在。

二，position:absolute

1. absolute的特性

  1. 破坏性： absolute元素脱离了文档结构，其他元素的位置重新排列了。只要元素会脱离文档结构，它就会产生破坏性，导致父元素坍塌（absolute绝对定位不仅让高度塌陷，又由于不在DOM tree上，所以宽度也是塌陷的）。
  2. 包裹性： 让元素inline-block化，宽度变成自适应内部元素的宽度。
  3. 跟随性： absolute元素 无依赖（没有可以依赖的relative父元素）情况下不设置top left 停留在static位置。

2.，使用时的知识点

  • 在不设置top left时，实现的是一定意义上的相对定位，相对定位之王，相对于原来位置，因为在static位置，设置margin是相对原来位置定位。
  • absolute元素会悬浮在页面上方，会遮挡住下方的页面内容。
  • absolute 去浮动，双重叠加生效的是absolute（除去IE7,会始终以元素inline-block位置，设置空div即可 ）。
  • 如果有多个悬浮元素，层级如何确定？答案是“后来者居上”.
  • 重叠效果：两个的话用文档流前后控制上下，多个的话用z-index:1控制，只有弹窗会大于2，要不然就是冗余.
  • absolute:left+right top+bottom会拉伸元素，替代高宽（IE7+支持）,同时使用时优先级是宽度高度更高。
  • 动画尽量用在绝对定位元素上，因为脱离了文档流这样就不影响其它元素。

三，position：fixed

其实fixed和absolute是一样的，唯一的区别在于：absolute元素是根据最近的定位上下文确定位置，而fixed永远根据浏览器确定位置。

float复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，float的概述

1. 基本概念

float一般的定义为浮动的框可以向左或向右移动，直到它的外边缘碰到包含框或另一个浮动框的边框为止。因此float一般被用来实现多列布局。但其实float最初的设计目的是为了实现文字环绕效果，所以具有了包裹性和破坏性两大特性。

2.float的破坏性

  • 被设置了float的元素会脱离文档流。（父元素高度坍塌，宽度不变的，但由于浮动元素仍在凡间DOM tree，父元素宽度没有坍塌）
  • 因为脱离了文档流，文档的普通流中的其它元素表现得就像浮动框不存在一样地正常排列，而由于float被设计的初衷——实现文字环绕效果,所以如果有文字(其实不止局限于文字，内联元素和内联块元素也是如此)是在float元素的覆盖范围内，即只要与float元素相接触时，会以环绕的方式排列在元素周围，甚至会脱离所在容器。
  • 破坏性会去空格化，并使元素block化，所以适合砌砖头。回车空格如果在标签末尾就不会显示。“清空格”这一特性的根本原因是由于float会导致节点脱离文档流结构。它都不属于文档流结构了，那么它身边的什么换行、空格就都和它没关系的，它就尽量的往一边去靠拢，能靠多近就靠多近，这就是清空格的本质。

3.float的包裹性

  • div设置了float之后，其宽度会自动调整为包裹住内容宽度，而不是撑满整个父容器。
  • 因为包裹性，浮动元素所在那一行的行框被缩短，从而给浮动框留出空间，行框围绕浮动。
  • 有多个浮动元素时，如下图所示，如果包含框太窄，无法容纳水平排列的三个浮动元素，那么其它浮动块向下移动，直到有足够的空间。如果浮动元素的高度不同，那么当它们向下移动时可能被其它浮动元素“卡住”：

二，清除float带来的影响–解决高度塌陷的问题

  • clear去浮动：通过在所有浮动元素下方添加一个clear:both的元素，可以消除float的破坏性。可以通过伪元素after实现。clear 属性规定元素的哪一侧不允许其他浮动元素。
  • 父元素包裹性去浮动：为父元素添加可以具有包裹性的css设置，如：overflow:hidden ，zoom:1 ，display:inline-block ，position:absolute ，float:left等。

一，盒子模型

1.w3c盒子模型

外盒尺寸计算（元素空间尺寸）:

Element空间高度 = content height + padding + border + margin

Element 空间宽度 = content width + padding + border + margin

内盒尺寸计算（元素大小）:

Element Height = content height + padding + border （Height为内容高度）

Element Width = content width + padding + border （Width为内容宽度）

2. IE盒子模型

外盒尺寸计算（元素空间尺寸）：

Element空间高度 = content Height + margin (Height包含了元素内容宽度，边框宽度，内距宽度)

Element空间宽度 = content Width + margin (Width包含了元素内容宽度、边框宽度、内距宽度)

内盒尺寸计算（元素大小）：

Element Height = content Height(Height包含了元素内容宽度，边框宽度，内距宽度)

Element Width = content Width(Width包含了元素内容宽度、边框宽度、内距宽度)

三，一些布局技巧

1. 水平居中

行内元素居中：
给父元素设置 text-align:center来实现的。

定宽块状元素居中：设置左右margin值为auto来实现居中的。

不定宽块状元素居中： 一共三种方法

  1. 第一步：为需要设置的居中的元素外面加入一个 table标签 ( 包括 <tbody>、<tr>、<td> )。第二步：为这个 table设置左右 margin居中（这个和定宽块状元素的方法一样）。
  2. 改变块级元素的 display 为 inline类型（设置为 行内元素 显示），然后使用 text-align:center 来实现居中效果。
  3. 通过给父元素设置 float，然后给父元素设置 position:relative 和 left:50%，子元素设置 position:relative和 left: -50%来实现水平居中。
  4. 绝对定位left:50%+margin-left:负自身一半。

2. 垂直居中

父元素高度确定的单行文本：设置父元素的 height和line-height 高度一致来实现的。

父元素高度确定的多行文本、图片等的竖直居中的方法有两种：

  1. 方法一：使用插入 table (包括tbody、tr、td)标签，同时设置 vertical-align：middle。css 中有一个用于竖直居中的属性 vertical-align，在父元素设置此样式时，会对inline-block类型的子元素都有用。
  2. 在 chrome、firefox 及 IE8 以上的浏览器下可以设置块级元素的 display 为table-cell（设置为表格单元显示），激活 vertical-align属性，但注意 IE6、7 并不支持这个样式, 兼容性比较差。

CSS基础知识复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，CSS的五个来源

  1. 因为浏览器默认样式的存在，我们需要为每个页面写上重置样式，简单重置样式如：* {margin:0; padding:0;}；复杂一点的网上有很多已经成熟的可以很好地处理浏览器兼容问题的重置样式文件可以引用,如：cssreset.com。
  2. 用户样式表，用户设置之后会覆盖浏览器的默认样式。
  3. <link>引用的CSS文件，外部式CSS样式，写在单独的一个文件中。
  4. <style>中编写的样式代码，嵌入式CSS样式，写在HTML文件中。
  5. <a style=’ ’>中编写的样式代码,内联式CSS样式，写在HTML标签中。

二，CSS之层叠

1. 层叠的概念

浏览器在解析CSS样式时，会将不同来源的CSS叠加在一起，再最终确定样式的结果。

2. 层叠的规则

  • 后三种样式是有优先级的：内联式 > 嵌入式 > 外部式,遵循就近原则（离被设置元素越近优先级别越高）。
  • 同一种样式中覆盖的默认规则是后者覆盖前者，即后写的覆盖先写的。
  • 上面所总结的优先级是在相同权值的情况下。权值指一个选择器的重要程度，标签的权值为1，类选择符的权值为10，ID选择符的权值最高为100。
  • 权值的两个特例：!important优先级最高，高于上面一切。*选择器最低，低于一切。
  • 还有一个权值比较特殊–继承也有权值但很低，有的文献提出它只有0.1，所以可以理解为继承的权值最低。

三，CSS之选择器

1. 标签选择器

  • 通用选择 *。
  • 单标签。
  • 多标签，主要有以下几种：子选择器（div>p），后代选择器(div p)，兄弟选择器（li+li），同胞选择器(span~a)。

    2.属性选择器

  • ID选择器
  • 类选择器
  • 只通过属性名选择和通过属性名和属性值选择

    四，伪类和伪元素

    1. 伪类

  • CSS 伪类用于向某些选择器添加特殊的效果。

2. 伪元素

  • CSS 伪元素用于将特殊的效果添加到某些选择器。

3. 二者区别

伪类的效果可以通过添加一个实际的类来达到，而伪元素的效果则需要通过添加一个实际的元素才能达到，这也是为什么他们一个称为伪类，一个称为伪元素的原因。 css3为了区分两者，已经明确规定了伪类用一个冒号来表示，而伪元素则用两个冒号来表示。

个人理解： 伪类是为使用伪类的选择器按照一定条件选出来的元素本身添加样式。伪元素是在使用伪元素的选择器选出来的元素基础上创建了一些新元素，进而为这些创建的元素添加样式。

HTML基础知识复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，HTML元素分类

1. 块状元素

  • 块状元素前后元素转行，自己独占一行。
  • 可以设置高度，宽度，以及顶部和底部边距。
  • 宽度默认为父元素宽度的100%。
  • 常见的块状元素有：<div>、<p>、<h1>...<h6>、<ol>、<ul>、<dl>、<table>、<address>、<blockquote> 、<form>。

2. 内联元素

  • 与其它元素在一行
  • 不可设置高度，宽度，以及顶部和底部边距。
  • 宽度由包裹的的内容物宽度决定
  • 常见的内联元素：<a>、<span>、<br>、<i>、<em>、<strong>、<label>、<q>、<var>、<cite>、<code>。

3. 内联块元素

  • 与其它元素在一行
  • 可以设置高度，宽度，以及顶部和底部边距。
  • 常见的内联块元素：<img>、<input>。

二，浏览器渲染页面

  1. 浏览器解析HTML构建DOM树，深度遍历，当前节点的子节点都构建完成之后才会去解析下一个兄弟节点。
  2. 解析CSS构造CSS规则树。
  3. 根据DOM Tree 和 CSS Rule Tree构建Rendering Tree 渲染树。
  4. Js通过 DOM API 和 CSSOM API 来操作 DOM Tree 和 CSS Rule Tree,执行绑定事件等，页面显示完成.

浏览器将载入的html变为dom树，但是此时没有任何显示样式。所有显示的样式，都是css定义的，浏览器只会通过css来渲染视图样式。
不要再说div天生就是block——这句话应该换成：浏览器默认样式天生规定了div是block——所以才导致了div是block！是浏览器的默认样式规定的，不是浏览器的内核规定的。

三，一些容易被忽略的知识点

  • HTML 文档 = 网页，HTML 元素指的是从开始标签（start tag）到结束标签（end tag）的所有代码。
  • 属性总是在 HTML 元素的开始标签中规定。
  • 当显示页面时，浏览器会移除源代码中多余的空格和空行。所有连续的空格或空行都会被算作一个空格。需要注意的是，HTML 代码中的所有连续的空行（换行）也被显示为一个空格。
  • div宽度和父容器相同，table宽度根据内容而定——即table具有“包裹性”。

  • <b></b>和<i>是物理元素,<strong></strong>和<em>是逻辑元素。

    物理元素所强调的是一种物理行为，比如说我把一段文字用b标记加粗了，我的意思是告诉浏览器应该给我加粗了显示这段文字，从单词的语义也可以分析得出，b是Bold(加粗)的简写，所以这个B标记所传达的意思只是加粗，没有任何其它的作用。

而Strong我们从字面理解就可以知道他是强调的意思，所以我们用这个标记向浏览器传达了一个强调某段文字的消息，而这个Strong就是我们所说的逻辑元素，他是强调文档逻辑的，并非是通知浏览器应该如何显示。

em是什么？——em是一个浏览器识别的长度单位，但是它不是绝对的、固定的，而是相对的。大家都知道px是一个绝对的长度单位制，它永远不会改变。浏览器默认情况下令1em === 16px。现在你知道为何p默认是16px了吧。而且你还知道了h1是p高度的两倍，h2是p高度的1.5倍……
