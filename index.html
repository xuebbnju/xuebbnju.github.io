js之对象与函数与对象复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，对象

1.概念

  • 若干属性的集合。属性时一对（key，value），key必须是字符串的。
  • 方法也是一种属性。因为它的属性表示为键值对的形式。
  • javascript中的对象可以任意的扩展属性，没有class的约束。

2. 创建

  1. 字面量 ：var obj={x:1;y:2}（这种方法创建的对象的原型默认指向object.prototype）。
  2. new构造器：prototype指向对象创建器的那个函数的prototype。
  3. obj.create ：Object.create(proto [, propertiesObject ]) 是E5中提出的一种新的对象创建方式，第一个参数是要继承的原型，如果不是一个子函数，可以传一个null，第二个参数是对象的属性描述符，这个参数是可选的。var obj=object.creat({x:1;y;2}),prototype是{x:1;y:2},最终都会指向object.prototype。

注意一点：如果，var obj=object.creat(null ),prototype就不能指向object.prototype，就无法用tostring等方法。

3. 属性操作

  • 属性读写：obi.x或obj[‘x’],或者for in（可能会访问原型链上的属性,顺序不确定）。
  • 属性删除：delete之后变为undefined，有些属性不允许被删除，有权限，比如object.prototype, 全局变量，局部变量，函数变量也都不可以被删除（隐式不用var定义的可以被删除，eval中的变量也可以）。
  • 属性检测：in 往原型链上查找。
    • .hasOwnPrototypr：对象上这个属性是否存在。
    • .prototypeIsEnumerable：属性是否可以枚举而输出，原型链上的大部分属性默认为false。
  • getter和setter方法：，get,set是关键字，自己写完方法后，读写属性时默认直接调用写的方法。
    • 对象原型链上有某个属性并且有getter和setter方法，对象本身没有这个属性时，对这个属性进行赋值操作不会像平常那样在这个对象内创建这个属性，而是调用set方法对原型链上的那个属性操作。
    • 即便没有getter和setter方法，若原型链上的那个属性writable为false，那么对象本身也无法添加这个属性。
    • 如果想在这个对象内部创建那个同名属性，只能通过definePropotry方法。

4. 属性标签

  • getOwnPropotryDescriptor方法获取属性标签。
  • object.definePropotry：增加属性并设置标签，是否可以枚举enumerable，可写writable等标签，默认为false。
  • 其他方法创建的属性的标签都默认为true。
  • object.key(obj),获取obj的所有属性的key。
  • object.definePropotries：定义多个属性。

5. 对象标签

  • proto标签
  • class标签：无法直接查看，可以通过Object.prototype.toString（即便参数是值类型也会转化成对象处理）获取。
  • extensible标签：是否可以扩展属性。
    • Object.isExtensible(obj)是否可以添加。
    • Object.preventExtensions(obj)阻止对象扩展属性，而不影响现有属性的标签。
    • Object.seal(obj)让对象所有属性configurable: false。Object.isSealed(obj);。
    • Object.freeze(obj)让对象所有属性configurable: false,, writable: false。Object.isFrozen(obj);。
    • 以上操作不影响原型链，除非遍历原型链把所有原型都设置一遍。

6. 序列化

  • JSON.stringify(obj)：把js对象转化为字符串（json格式）。
  • 属性值为undefine会把属性不存在处理，NaN,Infiity变null。
  • JSON.parse('{"x" : 1}')把字符串解析成js对象。
  • 可以为属性自定义toJSON方法。
  • 可以为对象自定义toString方法。
  • valeOf尝试把对象转换为基本类型是调用，也可以自定义重写。
  • 先找valeOf，之后找toString。

二，函数

1. 概念

  • 函数是一块JavaScript代码,被定义一次,但可执行和调用多次。
  • JS中的函数也是对象,所以JS函数可以像其它对象那样操作和传递所以我们也常叫JS中的函数为函数对象。
  • 函数就是包裹在花括号中的代码块，前面使用了关键词 function。

2. 局部变量与全局变量

  • 在 JavaScript 函数内部声明的变量（使用var）是局部变量，所以只能在函数内部访问它。（该变量的作用域是局部的）。
  • 您可以在不同的函数中使用名称相同的局部变量，因为只有声明过该变量的函数才能识别出该变量。
  • 只要函数运行完毕，本地变量就会被删除。
  • 在函数外声明的变量是全局变量，网页上的所有脚本和函数都能访问它。
  • 如果您把值赋给尚未声明的变量，该变量将被自动作为全局变量声明。
  • 无var的变量一律是全局的无论在哪写的。

3. 创建

  • 函数声明：fun add(){}。
  • 函数表达式：var add=fun(){} ;。
  • 命名函数表达式:var add=fun foo(){}。
    学习文章
  • 函数构造器：var func = new Function('a', 'b', 'console.log(a + b);');。
  • (fun(){})()：直接调用，立即执行。
  • 函数前面加括号和!,是为了让声明变成表达式，不被前置，且立即执行

4. 函数属性

  • .length：参数个数。
  • .name：函数名。
  • arguments,是一个类数组，没有join那些普通数组的操作函数，我的理解是，函数传进来参数之后，会自动创建一个arguments类数组来储存这些实参（严格模式下不成立）。
  • arguments.length:实参个数 。

js之执行上下文，this，闭包复习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，执行上下文

1.全局环境的准备工作（一条一条执行代码之前）

  1. 变量、函数表达式——变量声明，默认赋值为undefined；
  2. this——赋值；
  3. 函数声明——赋值；

2.局部环境的准备工作（函数体的语句执行之前）

  • 函数每被调用一次，都会产生一个新的执行上下文环境。

    1. 参数，arguments赋值。
    2. this赋值
    3. 局部变量声明，赋值undefined。

  • 当函数调用完成时，这个上下文环境以及其中的数据都会被消除，再重新回到全局上下文环境。

  • 函数在定义的时候（不是调用的时候），就已经确定了函数体内部自由变量的作用域。

3. 执行上下文栈

  • 处于活动状态的执行上下文环境只有一个。
  • 未调用函数时，处于全局的上下文环境，执行每行代码做的是赋值工作。
  • 执行函数体语句之前，会创建一个新的执行上下文环境（为函数体内的局部变量做准备工作），并将这个执行上下文环境压栈，设置为活动状态。

二，作用域

1. 基础概念

  • javascript除了全局作用域之外，只有函数可以创建的作用域。
  • 作用域有上下级的关系，上下级关系的确定就看函数是在哪个作用域下创建的。
  • 作用域最大的用处就是隔离变量，不同作用域下同名变量不会有冲突。

2. 作用域链

  • 自由变量： 在A作用域中使用的变量x，却没有在A作用域中声明（即在其他作用域中声明的），对于A作用域来说，x就是一个自由变量。
  • 要到创建这个函数的那个作用域中取值——是“创建”，而不是“调用”。
  • “作用域链”过程：
    1. 第一步，现在当前作用域查找a，如果有则获取并结束。如果没有则继续；
    2. 第二步，如果当前作用域是全局作用域，则证明a未定义，结束；否则继续；
    3. 第三步，（不是全局作用域，那就是函数作用域）将创建该函数的作用域作为当前作用域；
    4. 跳转到第一步。

三，作用域与上下文环境

  • 作用域在函数定义时就已经确定了。而不是在函数调用时确定。
  • 作用域只是一个“地盘”，一个抽象的概念，其中没有变量。要通过作用域对应的执行上下文环境来获取变量的值。
  • 同一个作用域下，不同的调用会产生不同的执行上下文环境，继而产生不同的变量的值。
  • 作用域中变量的值是在执行过程中产生的确定的，而作用域却是在函数创建时就确定了。
  • 如果要查找一个作用域下某个变量的值，就需要找到这个作用域对应的执行上下文环境，再在其中寻找变量的值。

一些重点： 一个作用域下可能包含若干个上下文环境。有可能从来没有过上下文环境（函数从来就没有被调用过）；有可能有过，现在函数被调用完毕后，上下文环境被销毁了；有可能同时存在一个或多个（闭包）。

四，闭包

  1. 函数作为返回值。
  2. 函数作为参数被传递。

核心：作为返回值或者参数的函数，会创建一个作用域，而且需要使用一个不属于这个函数内部的自由变量，这使得这个自由变量所在的函数即便已经执行完了，它的上下文环境也不能被销毁，还需要在上下文栈中，但不是作为活动上下文环境存在。所以使用闭包会增加内容开销。

五，this

1.普遍的五种情况

情况1：构造函数(new对象的函数)具体构造出的哪个对象调用自己this就指谁。

情况2：函数作为对象的一个属性 函数中的this指向该对象。

情况3：函数用call或者apply调用 this的值就取传入的对象的值。

情况4：全局 & 调用普通函数 在全局环境下，this永远是window。

情况5：不仅仅是构造函数的prototype，即便是在整个原型链中，this代表的也都是当前对象的值。

2. 一些特例

  • 函数定义在一个对象的属性函数内，它仍然是一个普通的函数，this仍然指向window。
  • 一个对象的函数属性被赋值给一个变量，然后通过这个变量调用函数时this仍然指向window。

3.总结

  • 除非这个函数明确是某个对象属性或者是作为函数构造器构造为对象之后调用，都是指向window，与定义在哪里的位置无关。
  • 函数体有点像C++中的静态函数，不管哪个引用指向此函数体，函数体的内容都是一样的，执行语句一样，只是this不一样，传进来的参数不一样导致了区别，所以函数赋值只是赋给引用，并不是全部复制。

一，函数和对象的关系

  • 对象都是通过函数创建的。
  • 而函数却又是一种对象。

二，prototype原型

  • 每个函数都有一个属性叫做prototype。
  • prototype的属性值是一个对象（属性的集合，再次强调！），默认的只有一个叫做constructor的属性，指向这个函数本身。
  • 每个对象都有一个proto，可称为隐式原型，指向创建该对象的函数的prototype。

以上几条达到的结果是：Fn是一个函数，fn对象是从Fn函数new出来的，这样fn对象就可以调用Fn.prototype中的属性。

三，环形结构

  • 自定义函数的prototype本质上就是和 var obj = {}是一样的，都是被Object创建，所以它的__proto__指向的就是Object.prototype。
  • Object.prototype确实一个特例——它的__proto__指向的是null。
  • 函数也是一种对象，函数也有__proto__，Function创建了函数,所以指向Function.prototype。
  • Function.__proto__指向Function.prototype。
  • Function.prototype指向的对象，它的proto也指向Object.prototype

父类

1. 原型链继承

实现原理: 将父类的实例作为子类的原型。

2.构造继承

实现原理: 使用父类的构造函数来增强子类实例，等于是复制父类的实例属性给子类（没用到原型）

  1. change事件，这个事件不太友好，触发必须满足两个条件：a）当前对象的值发生改变，并且是由
    键盘或鼠标事件激发的（脚本触发无效）；b）当前对象要失去焦点(onblur)

  2. keypress事件，这个一般，对比上面的要好，就是只能监听键盘事件，鼠标复制黏贴等操作就无能为力了

  3. propertychange和input事件，这两个事件其实是一回事，只是不同的标准罢了，可以同时绑定。

    • input是标准的浏览器（W3C）事件，一般应用于input元素，当input的value发生变化就会发生，无论是键盘输入还是鼠标,重点是黏贴的改变都能及时监听到变化；
    • propertychange是IE专属的事件，只要当前对象属性值发生改变就能触发。

6. event和this

  • event.targe是实际的触发事件的元素，而this是给哪个元素绑定了事件。
  • event.target代表当前触发事件的元素，可以通过当前元素对象的一系列属性来判断是不是我们想要的元素。
  • js中事件是会冒泡的，所以this是可以变化的，但event.target不会变化，它永远是直接接受事件的目标DOM元素；
  • this和event.target都是dom对象。
  • 如果要使用jquey中的方法可以将他们转换为jQuery对象。比如this和$(this)的使用、event.target和$(event.target)的使用；

  • 点击发射按钮时为野狗云实例message子节点添加键值对，同时将输入框中的内容清空。
  • 按回车键时，通过trigger("click");触发发射按钮点击事件。
  • message子节点监听到添加键值对事件后，将添加的键值对的值内容取出压入保存弹幕内容的数组中。

  • 创建以此值内容为文本的弹幕div,并添加到弹幕框div中(注意要将DOM对象转化成jQuery对象):

  • 然后将此弹幕div实现运动。

    • 由于每次新添加的弹幕div只会跑一次，为了让弹幕框里一直有弹幕滚动，设置一个定时器函数，每个一段时间就从保存弹幕内容的数组中随机抽一个弹幕出来，添加弹幕div让它实现移动。
  • 点击清屏按钮时调用ref.remove();清空所有键值和子节点。并情况保存弹幕内容的数组。

2. 弹幕运用原理

  3. 为弹幕添加动画，通过目标值设为左偏移为负的自身宽度，让弹幕div从左到右运动一次。并随机添加颜色和时间设置。
  4. 随机时间通过生产随机数实现，随机颜色通过从0-9和a-f中随机选取一个字符出来，随机地选6次组成颜色字符串。
  5. 动画的回调函数需要设置成删除此div节点，否则一直存在，浪费空间。

QQ面板实例练习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，基础知识

1. 事件流

  1. 事件冒泡:
    事件冒泡，即事件最开始由最具体的元素(文档中嵌套层次最深的那个节点)接收，然后逐级向上转播至最不具体的节点(文档)。
  2. 事件捕获:
    事件捕获的思想是不太具体的节点应该更早接收到事件，而最具体的节点最后接收到事件。

2. 事件处理程序

  1. 在html代码元素中添加事件为Html事件。
  2. 使用元素的onclick方法为DOM0级事件处理。
  3. 使用事件添加方法为元素添加事件:

3. 事件对象

兼容问题:e=e || window.event;

  1. DOM中的事件对象
    • type:获取事件类型
    • target：事件目标
    • stopPropagation() 阻止事件冒泡
    • preventDefault() 阻止事件的默认行为
  2. IE中的事件对象
    • type:获取事件类型
    • srcElement：事件目标
    • cancelBubble=true阻止事件冒泡
    • returnValue=false阻止事件的默认行为

二，拖拽与切换状态

1. background-position 学习文章

  • 位置设定是指图片与坐标原点的偏移量
  • 原点是不会动的，移动的是图片，超出容器的部分隐藏。
  • X坐标为正则图片左上角向右平移，为负则图片左上角向左平移。
  • Y坐标为正则图片左上角向下平移，为负则左上角向上平移。
  • 百分比的计算是有公式的：X轴( 容器宽度 – 图片宽度 )*含符号百分比。
  • Y轴( 容器高度 – 图片高度)*含符号百分比。

2. 拖拽原理

  • 监听onmousedown事件。

  • 在监听函数中计算出出发时鼠标离面板左上角的上下距离。

  • 然后再在函数中添加两个事件监听onmousemove和onmouseup。

  • 触发onmousemove事件意味鼠标在移动，获取onmousemove事件鼠标的位置。
  • 将鼠标位置减去与面板左上角的距离之后就是面板所在位置，通过样式设定left和top定位即可。
  • 因为鼠标可以移动到角落，会导致面板位置超出窗口可视区域，需要将位置合理化，具体如下：
    * 面板左上角横坐标小于0都设为0，面板横坐标过大导致面板右侧不可见时，设为最大值（window宽度减去面板宽度）
* 面板左上角纵坐标坐标小于0都设为0（有时因为边距之类的问题需要比0大一点），面板纵坐标过大导致面板底部不可见时，设为最大值（window高度减去面板高度）。
  • 触发onmouseup事件意味着停止拖拽，此时清除监听onmousemove（赋值为null），因为此时鼠标移动时就不拖拽了。

3. 切换状态原理

1.样式设置要点:

  • 显示的状态图标和状态文字div要同时有类名和id,类名用于设置所有状态图标和文字的通用样式，id用来获取div。
  • 列表中的，li包含了备选的状态图标和状态文字div,为了在获取的时候区分状态和文字要添加不同的类名。
  • 列表中的状态图标和显示的状态图标div都要一个标志自己的类名用来设置自己独有的样式。
  • 为了方便获取每个状态图标的独特类名，将它的父元素li的id和这个类名设置成一样的。

2.实现原理:

  • 点击状态区域，备选状态列表diplay:block。
  • 遍历列表，对每种状态的li添加监听事件。
  • 触发onmouseover事件，li背景色改变，凸显鼠标滑过效果。
  • 触发onmouseout事件，li背景色改为本来颜色。
  • 触发点击事件，获取这个状态li下的状态图标的独特类名(通过父元素id),然后将显示状态的div的独特类名改为这个选中的li下的状态图标的独特类名。stateShow.className='login-state-show '+id;
  • 获取这个状态li下的状态文字，将显示状态文字的div的内容改写为这个选中的li下的状态文字。stateTxt.innerHTML=getByClass('stateSelect_text',id)[0].innerHTML;
  • 给整个文档绑定点击事件触发之后状态备选列表display:none消失。（其它绑定了点击事件的元素被点击之后因为事件冒泡会触发文档元素的点击事件，这是我们所不希望看到的，所以要阻止事件冒泡）。

切图实例练习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，样式设置

  1. 所有装照片的div宽度一致，高度auto。
  2. 所有装照片的div会设置为absolute,所以父元素需要relative。
  3. 计算出可以放置的列数，然后设置最外层盒子的宽度。

二，实现思路

  1. 遍历所有的照片div，对它们进行定位设置。
  2. 建立一个数组，储存第一排元素的outerHeight,作为每一列的高度初始值.
  3. 从第二排的第一个照片div开始，找出现在高度最小的那一列，便通过设置样式将元素放在这一列。
  4. 更新这一列的高度值。
  5. 储存每列高度值的数组需要在循环函数之外定义，否则会成为循环函数的内部变量，每次都会重新定义赋值。
  6. 当滑动到最后一个照片一半的时候，创建一排新的照片div，然后循环以上步骤。

三，各种高宽值区别

  1. 通过style.width(style.height)或者jQuery的$(divObj).width()/$(divObj).height()获取/设置元素的宽高
  2. 若要获取元素包含边框的宽度，则可通过divObj.offsetWidth/divObj.offsetHeight或jQuery的$(divObj).outerWidth(false)/$(divObj).outerHeight(false)获取
  3. 通过$(divObj).outerWidth(true)/$(divObj).outerHeight(true)获取带外边距的宽度

  4. 通过$(divObj).innerWidth()/$(divObj).innerHeight()获取不包含边框、不包含外边距的宽度

物体运动案例练习总结

声明：

所含内容是从网上的资料总结而来，并非原创，所含文字图片侵权必删！

一，单物体单一属性变化

1. 思路

  1. 通过setInterval()函数实现物体属性连续变化。
  2. 每次调用都会对当前属性进行增加（减少）操作。
  3. 当前属性达到目标值时清除定时器clearInterval()。

2. 注意事项

  • 每次调用setInterval()函数前先clearInterval()，否则多次触发事件会激活多个定时器。
  • 普通属性读取时利用offset（获得数值）,写时就直接赋值需要加单位变字符串。
  • 速度根据目标值与当前值的大小关系确定正负。
  • 为了实现缓冲效果，可以动态地设置速度为目标值与当前值之差的某个倍数（需要取整，否则无法正好达到目标值）。

3.特殊的透明度属性

  • 由于无法直接获取当前取值，需要手动用一个变量记录当前值。
  • 特殊的赋值方式：oDiv.style.filter='alpha(opacity:'+alpha+')';
  • 赋值还需要考虑兼容性问题。

二，多物体运动单一属性变化

  • 思路与单物体运动一样，只是需要每个物体有单独的定时器，否则会混淆。
  • 透明度变化时，也需要为每个物体设置一个储存当前值得变量不可共用。

三，单一属性变化通用函数

  • 把要变化的属性作为参数传入。

  • 通过通用的获取样式方法获取此属性的当前值。(浏览器兼容)

  • 其它操作如常，只是透明度需要单独判断操作（因为有些是浮点数，还有百分比转化无法直接取值和赋值）。

四，同时运动（多个属性同时变化）

  • 多个属性和目标值以json格式作为参数传入。
  • 设置定时器函数，每隔一段时间就遍历一次属性并同时进行加减操作（这样才实现了同时）。
  • 通过设置flag,三个属性都到达目标值时清除定时器。

五，链式运动（多个属性接连变化）

  • 增加一个回调函数作为最后一个参数。
  • 每个属性变化完之后会判断是否还有需要变化的属性，有的话就继续调用，没有就结束。
